1 Hierarchy Generation
2 ====================
3 This is a TMP pattern to generate a class hierarchy from a collection of policy classes. It permits arbitrarily mixing policies to generate rich components while avoiding multiple inheritance. It works by chaining together a series of policies in a linear class hierarchy through a couple controller templates.
4 
5 The various policies are required to implement a parameterized super class idiom:
6 ```{.cpp}
7 template <class _SuperT> class OnClick : _SuperT {
8  ...
9 };
10 template <class _SuperT> class OnPaint : _SuperT {
11  ...
12 };
13 template <class _SuperT> class OnResize : _SuperT {
14  ...
15 };
16 template <class _SuperT> class OnTimer : _SuperT {
17  ...
18 };
19 ```
20 Each policy template is a self contained behavioral unit that describes an aspect of the larger whole. To maximize reuse they should be independent but it's possible to share information between them if care is taken.
21 
22 The process of chaining the policies together is performed through a controller with the following declaration:
23 
24 ```{.cpp}
25 template <class _BaseT, template <class> class ... _PolicyList>
26 class HierarchyGenerator;
27 ```
28 
29 The parameters of the `HierarchyGenerator` are `_BaseT` and `_PolicyList`. The `_BaseT` parameter in this example is the super class of the entire hierarchy and will normally be a non-templated class. Policies are passed as parameters to the `HierarchyGenerator` in the `_PolicyList` parameter to successively declare a linear class hierarchy:
30 
31 ```{.cpp}
32 class Window{
33 ...
34 };
35 using Button = HierarchyGenerator<Window, OnClick, OnPaint, OnResize>;
36 using Label = HierarchyGenerator<Window, OnPaint>;
37 using Sizer = HierarchyGenerator<Window, OnClick, OnPaint, OnResize, OnTimer>;
38 using Timer = HierarchyGenerator<Window, OnTimer>;
39 ```
40 This results in linear class hierarchies composed of a mixture of policies. It equivalently declares the following:
41 
42 ```{.cpp}
43 using Button = OnClick<OnPaint<OnResize<Window>>>;
44 using Label = OnPaint<Window>;
45 using Sizer = OnClick<OnPaint<OnResize<OnTimer<Window>>>>;
46 using Timer = OnTimer<Window>;
47 ```
48 The two forms are the same and the varation using the `HierarchyGenerator` is more verbose but the intent is clearer and more readable.
49 
50 The 'magic' is done through a couple partial specializations of `HierarchyGenerator`:
51 ```{.cpp}
52 template <typename _BaseT> class HierarchyGenerator<_BaseT> : public _BaseT{
53  template <typename ... _arg_ts> HierarchyGenerator(_arg_ts&&...oArgs) : _BaseT(std::forward<_arg_ts>(oArgs)...){}
54 };
55 ```
56 This is the final specialization in the recusive template instantation chain that makes `_BaseT` the base class of all the policies. The compiler selects this specialization when the `_PolicyList` has been depleted of items. Chaining together the items in the `_PolicyList` is performed with the following partial specialization:
57 ```
58 template <typename _BaseT, template <class> class _PolicyT, template <class> class ... _PolicyList>
59 class HierarchyGenerator<_BaseT, _PolicyT, _PolicyList...> : public _PolicyT< HierarchyGenerator<_PolicyList...>> {
60  template <typename ... _arg_ts> HierarchyGenerator(_arg_ts&&...oArgs)
61  : _PolicyT< HierarchyGenerator<_PolicyList...>>(std::forward<_arg_ts>(oArgs)...){}
62 };
63 ```
